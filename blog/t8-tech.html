Expressing the Regulars

Technical Blog
29 MAR 15

So what is a regular expression (aka 'regex')? A combination of characters used to define a search pattern. Regular expressions are also often used to check formatting (passwords, email input, phone number input, activation codes, etc...).

Regexes (yes, 'regexes') make it easy to create a search parameter without needing multiple lines of code. It also acts as a single search rather than multiple searches.

An example of a regex used to validate an email address:
\b[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,4}\b
(source: http://www.regular-expressions.info/email.html)

In the email validation, above, there a sections that define the search parameters to match, and how many characters to allow before the next section. (An interactive site that explains each section when hovered over : http://www.regexbuddy.com/regex.html)

Regexes can look hard and confusing, so it's best to know exactly what is required for the parameters before implementing one into the code.
