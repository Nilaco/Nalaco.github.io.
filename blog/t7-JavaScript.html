How does programming promote laziness? Looping! Using loops allows you to go over a section over a section code any number of times without having to retype it. The concept of looping is pretty much the same, however the syntax differs between Ruby and Javascript (if you want it to). There are multiple types of loops to use, the most common are as follows:

For Loops: Loop through the code block once every time an element in the expression is reached. For example:

for i in 0..10

(Ruby) will run the code every for every element in 0..10, which will temporarily be assigned to "i". The result would be every number from 0..10. In Javascript the loop functions the same way, however, initialing can be seen as

for (i = 0; i <= 10; i++)

(JS). In JS, all the parameters can be defined in the initial statement. i = 0, sets the starting counter. i <= 10, when this is met, the looping ends. i++, after every run of code, add 1 to the initial counter.

While Loops: Loop through the code block WHILE the condition is TRUE. While loops will continually go through whatever code is held within, until the initializing expression becomes false.

while survived = false

will run the loop as long as survived stays false. As soon as survived becomes true, the code will not run. JS also checks while loops the same while

while (i < 15)

. With the both examples, you need to make sure that there is something in the code that will eventually change the initial expression to FALSE. Otherwise the while loop will never end. Which is not a good thing.

Ruby also has an 'until' loop which is pretty much the opposite of while. It will loop through the code while the expression is false, or UNTIL the expression is TRUE.
